## 五、失败组别与业务影响

### A. 主旨题自动定向修复没有按预期触发

代表失败：

- `tests/test_question_generation.py::QuestionGenerationUnitTest::test_main_idea_axis_errors_enable_targeted_quality_repair_even_on_compact_path`

问题含义：

- 当系统已经识别出主旨题存在“主轴偏了、抽象层级偏了”这类错误时，本来应该触发一次“定向质量修复”
- 现在这条自动补救链没有按预期起效

业务影响：

- 系统自愈能力会变弱
- 某些主旨题如果第一次生成得不够准，系统不一定能自动拉回来
- 但这**不会直接导致按钮不可用，也不会导致工作台链路断掉**

影响等级：

- `中`

上线判断：

- `不阻断这次工作台上线`
- 但会影响“自动质量修复能力”的可信度

---

### B. 中心理解 / 主旨类 validator 的语义判定不够稳

代表失败集中在：

- `tests/test_question_validator.py`

涉及的失败轴包括：

- `main_axis_mismatch`
- `local_point_as_main_axis`
- `example_promoted_to_main_idea`
- `abstraction_level_mismatch`
- `argument_structure_mismatch`

问题含义：

- validator 本该识别“把局部点当主旨”“把例子拔成中心”“抽象层级不对”“论证结构判断偏了”等问题
- 但当前没有稳定打出这些预期错误

业务影响：

- 主旨题有可能“看起来像过了系统检查”，但实际上中心抓得不够准
- 如果业务完全依赖自动 validator 放行，会有漏检风险
- 好在你当前的产品设计里，`确认通过` 是人工最终裁决，审核页也允许人工修题，因此这部分风险被人工环节兜住了一部分

影响等级：

- `高`

上线判断：

- `不阻断本次工作台上线`
- 但阻断“对外宣称主旨题自动质量判断已经很稳”

---

### C. 语句填空题 validator 对功能位、承前启后、指代锚点的判断不够稳

代表失败也集中在：

- `tests/test_question_validator.py`

涉及的典型问题包括：

- 开头句 / 中间桥接句 / 结尾对策句 功能识别不稳
- `bidirectional_validation` 没按预期约束住
- `reference_anchor / reference_dependency` 支撑不足
- 原句删除后的改写与落点约束不稳

问题含义：

- 系统对于“这道填空题到底是不是一个真卡口”“是不是既要承前又要启后”“是不是需要强指代对齐”这类判断还不够稳

业务影响：

- 会直接影响填空题质量
- 可能出现“题能做出来，但卡口不够准、干扰项不够像、正确性边界不够清楚”
- 如果依赖自动 validator 兜底，存在漏判

影响等级：

- `高`

上线判断：

- `不阻断这次工作台按钮和流程上线`
- 但会影响填空题自动质量保障能力

---

### D. 排序题 schema / 合同层与预期存在漂移

代表失败：

- `tests/test_sentence_order_schema.py`

涉及问题包括：

- 句子数量合同不一致
- `sentence_count_mismatch` 没按预期打出
- `role_order_conflict` 没按预期打出
- 资源配置和 schema 预期存在偏移

问题含义：

- 排序题在“结构合同层”还有资产和规则不一致的问题

业务影响：

- 排序题可能出现结构约束不够严
- 某些错误排序逻辑没有被系统及时识别
- 更像是“排序题内核质量控制还有债”，不是页面工作台功能坏了

影响等级：

- `高`

上线判断：

- `不阻断工作台交互上线`
- 但会影响排序题质量控制的稳定性

---

### E. 旧分支兼容残留：continuation / title_selection / 旧 validator 合同

代表失败仍主要在：

- `tests/test_question_validator.py`

问题含义：

- 仓里还残留了一些旧分支、旧命名、旧合同兼容逻辑
- 例如 `title_selection` 这一类历史遗留概念在 validator 或配置层还有残存预期

业务影响：

- 主要会增加系统维护复杂度
- 容易让老配置、老测试、旧接口口径互相打架
- 对你这次已经清过的用户主流程，影响比前面几组小

影响等级：

- `中`

上线判断：

- `不阻断本次上线`
- 但后续最好继续清历史债

## 六、这些遗留失败会不会影响你这次要上的页面和按钮

### 直接答案

`不会把这次已经验过的工作台主链路打断。`

原因是：

- 本次上线关注的是“用户能不能顺畅建题、审核、修题、确认、追溯、下载”
- 这条链我已经做了定向测试和真实点击验证
- 现存失败更多集中在“自动质量判断内核”和“某些题型 validator / schema 的历史债”

### 但它们会影响什么

它们主要影响的是：

- 系统自动判断题目质量时是否足够准
- 某些题型的自动修复和自动校验是否足够可信
- 是否适合对外宣称“自动质量内核已经非常稳定”
