# 各环节与各题卡蒸馏实施表

## 1. 服务环节拆分

结合你现在的系统，建议按下面 6 个环节做蒸馏。

1. 原始真题接入
2. 材料处理
3. 材料检索与映射
4. 题目生成
5. 审核评分
6. 用户行为反哺

每一环都单独评测，不要把问题都推给生成模型。

## 2. 环节级实施

## 2.1 原始真题接入

目标：

- 把真题变成稳定可回放的标准样本

要做的事：

- 统一材料、题干、选项、答案、解析字段
- 清洗 OCR 和格式噪声
- 建立题型规范名
- 建立真题唯一 ID

交付：

- 真题标准库 v1

## 2.2 材料处理

目标：

- 把“可读”改成“可出题”

要做的事：

- 建立两条处理链：阅读辅助型 / 主旨保持型
- 用真题回放比较“处理前后是否仍能支撑原题”
- 记录每次处理丢失的信息类型

核心指标：

- 原题可还原率
- 逻辑链完整率
- 脏文本清洗率
- 主旨偏移率

## 2.3 材料检索与映射

目标：

- 先拿到真正适合该题卡的材料

要做的事：

- 对齐 `business_family_id`
- 对齐 `question_card_id`
- 对齐 `business_card_id`
- 对齐 `material_structure_label`
- 对齐 `slot_position / function_type`

核心指标：

- top1 命中率
- top3 命中率
- 错卡率
- 结构错位率

## 2.4 题目生成

目标：

- 让成题更像真题，不像随手拼接

要做的事：

- 做题干风格库
- 做错项风格库
- 做解析风格库
- 做按题卡 few-shot 检索

核心指标：

- 正确项抽象层级匹配率
- 错项错法匹配率
- 解析结构完整率
- 人工最小修改率

## 2.5 审核评分

目标：

- 让自动评分更可信

要做的事：

- 用人工金标校准评分
- 重新定义继续复核阈值
- 给不同题卡建立不同评分容忍区间

核心指标：

- 自动评分与人工评分相关性
- 误杀率
- 漏放率

## 2.6 用户行为反哺

目标：

- 学会“什么题真有用”

要做的事：

- 分析下载题
- 分析用户改题 diff
- 分析被反复丢弃的题

核心指标：

- 下载率
- 下载后修改率
- 高频修复字段

## 3. 题卡级实施

## 3.1 `main_idea / center_understanding`

现状特点：

- 最适合开放主旨保持型创作
- 最容易暴露抽象层级和主轴错位问题

优先蒸馏内容：

1. 主轴来源
2. 抽象层级
3. 覆盖范围
4. 错项错法

优先数据：

- 真题主旨概括
- 标题选择
- 结构概括

先做法：

- 从真题中抽 `main_axis_source`
- 抽正确项抽象层级
- 抽 3 个最常见错误项错法
- 让生成时按卡检索这些案例

## 3.2 `sentence_fill`

现状特点：

- 最依赖结构位置和功能类型
- 也是最容易因为映射错而整体崩掉的卡

优先蒸馏内容：

1. opening / middle / ending 题位
2. function type
3. bidirectional dependency
4. 错项的功能错位方式

建议按子卡拆：

- `opening_summary`
- `opening_topic_intro`
- `middle_bridge_both_sides`
- `middle_lead_next`
- `middle_carry_previous`
- `ending_summary`
- `ending_countermeasure`

先做法：

- 每个子卡先收 30-50 个真题样本
- 抽“正确项为什么对”的功能说明
- 抽“错误项为什么错”的功能偏移模板
- 对检索环节先做 top-k 回放

## 3.3 `sentence_order`

现状特点：

- 很能检验材料结构链是否真实存在
- 材料稍微洗坏就很难做

优先蒸馏内容：

1. 句间线索类型
2. 首尾锚定方式
3. 指代链
4. 时间/逻辑/总分结构链

建议按子卡拆：

- `head_tail_lock`
- `head_tail_logic`
- `discourse_logic`
- `timeline_action_sequence`
- `role_chain`

先做法：

- 从真题抽“排序证据类型”
- 对材料处理链做强约束，不允许过度改写
- 建一个排序证据词典

## 3.4 `continuation`

现状特点：

- 更偏生成，不适合过早做深蒸馏
- 可以放在第二阶段

优先蒸馏内容：

- 语气延续
- 叙述视角
- 主题延续
- 风格一致性

## 4. 按环节的第一轮动作

如果你只准备先做一轮试点，我建议：

### 试点包 A：`main_idea`

- 200 题真题
- 先做主轴、抽象层级、错项错法
- 输出主旨卡蒸馏 prompt 包

### 试点包 B：`sentence_fill`

- 200 题真题
- 先做题位、功能类型、错项功能偏移
- 输出填空卡映射校准表

### 试点包 C：评分校准

- 选 100 题人工打分
- 对自动评分做一次回归校准

## 5. 每轮产出物

每做完一轮，至少产出这 6 个东西：

1. 蒸馏样本包
2. 标注口径文档
3. 版本前后对比报表
4. 错误样本清单
5. 新增/修订的题卡映射表
6. 是否上线建议

## 6. 明确不建议先做的事

现阶段先不要：

- 直接微调大模型
- 线上自动改 prompt
- 让用户行为数据直接改分数阈值
- 不分题卡地全量混合蒸馏

这样会把口径越做越乱。

## 7. 建议你马上启动的顺序

1. 先整理 `main_idea` 真题样本
2. 再整理 `sentence_fill` 真题样本
3. 用这两类样本跑一次完整回放
4. 再决定先修“材料处理”还是先修“映射检索”

这个顺序最容易快速看到提升，也最容易定位问题归因。
