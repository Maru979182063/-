# 真题驱动蒸馏总计划

## 1. 目标

这套计划不是直接把真题拿去“训练一个更大的模型”，而是先用真题反向打磨现有系统的 5 个关键层：

1. 材料处理层  
   对齐“强原文题型”和“主旨保持型题型”的材料改写边界。
2. 材料检索与映射层  
   对齐题型、业务卡、材料卡、结构标签、题位功能之间的映射关系。
3. 题目生成层  
   对齐题干风格、错误选项错法、解析结构、难度控制。
4. 审核与评分层  
   对齐质量分、难度分、处罚分、继续复核阈值。
5. 自适应优化层  
   用“用户下载题”“用户修改后下载题”“生成失败题”做离线回放优化。

这五层打顺以后，题卡会明显顺滑，主要体现在：

- 题卡命中更稳
- 错题不像乱生成
- 分数更贴近业务体感
- 材料不会总是“能读但不能出题”
- 审核页的修复动作会更有抓手

## 2. 现阶段原则

### 2.1 不做线上自动自学习

先记录，再离线验证，再发布。  
不要让线上失败样本直接回写规则或提示词。

### 2.2 先蒸馏规则和映射，再考虑训模型

第一阶段优先做：

- 真题金标准集
- 题卡映射校准
- 材料处理边界校准
- 候选材料排序校准
- 错项风格库
- 难度与质量分校准

第二阶段再做：

- 小分类器
- reranker
- 质量判别器
- 难度预测器

### 2.3 一切围绕“回放可验证”

每一次规则调整、提示词调整、映射调整，都必须能回放到一批真题集上比较前后效果。

## 3. 分阶段实施

## 阶段 A：真题基线库

目标：建立一套可靠的“评测母库”。

建议至少拆成 4 个集合：

1. 材料处理基线集  
   原文材料 + 真题题干 + 选项 + 答案 + 解析
2. 题卡映射基线集  
   每题对应题型、业务卡、结构卡、材料结构标签、错误选项错法
3. 难度校准基线集  
   每题人工给出业务难度、可用性、迷惑度、主旨显隐度
4. 失败对照集  
   线上生成失败题、低分题、被用户改动较大的题

阶段 A 的产出：

- `gold_truth_questions`
- `gold_material_processing_cases`
- `gold_card_mapping_cases`
- `gold_scoring_cases`

## 阶段 B：材料处理蒸馏

目标：把“材料能读”和“材料能出题”区分开。

这里单独做两条链：

1. 阅读辅助型处理  
   仅用于强原文题型，如语句填空、语句排序。
2. 主旨保持型处理  
   仅用于中心理解类，允许围绕主旨进行合理创作。

核心判断维度：

- 信息是否丢失
- 逻辑链是否被改断
- 小标题、编号、说明性脏文本是否清干净
- 改写后还能不能稳定还原真题出题依据

## 阶段 C：映射与检索蒸馏

目标：让系统先拿对材料，再谈生成。

优先校准：

1. 题型 -> business family
2. 题型 -> question card
3. question card -> business card
4. business card -> material structure label
5. sentence_fill function type 规范化映射
6. opening / middle / ending 题位与结构信号对齐

阶段 C 不追求一次全自动，先让“高风险卡”稳定下来。

## 阶段 D：题目生成蒸馏

目标：让模型不再“凭空想题”，而是“参照真题风格成题”。

重点蒸馏 4 件事：

1. 题干风格
2. 正确项抽象层级
3. 错项错法分布
4. 解析结构

蒸馏手段：

- 案例检索增强
- 结构模板抽取
- 错项风格词典
- 解析模板词典

## 阶段 E：评分与审核蒸馏

目标：让系统分数和人工体感尽量接近。

优先校准字段：

- overall_score
- material_alignment
- difficulty_fit
- distractor_quality
- abstraction_match
- risk_penalties
- recommended_state

建议先做一个轻量版本：

- 基于真题和人工审核结果做分段规则校准
- 再做一个小型 rerank / scorer

## 阶段 F：用户行为反哺

目标：让系统学会“什么题真被拿走了”。

可利用数据：

- 用户下载的题
- 用户修改后下载的题
- 用户丢弃的题
- 高频触发“参数重做”的题
- 审核页常改的错误选项类型

这一层只做离线分析，不直接回写线上。

## 4. 推荐优先级

如果你想尽快看到效果，建议按这个顺序做：

1. `center_understanding / main_idea`
2. `sentence_fill`
3. `sentence_order`
4. 评分与审核校准
5. 用户行为反哺

原因：

- 中心理解最适合先建立“主旨保持型创作”的边界
- 语句填空最依赖结构和题位，是检索映射问题最暴露的地方
- 语句排序能很好检验材料结构链是否真实可出题

## 5. 预计交付节奏

### 第一轮：两周可见效果

- 建金标准样本库
- 做 main_idea + sentence_fill 两卡
- 补错项风格库
- 补评分校准报表

### 第二轮：一月内形成闭环

- 覆盖 sentence_order
- 建立离线回放脚本
- 建立版本对比报表
- 用用户行为样本加入二次校准

## 6. 成功标准

计划是否有效，不看“模型觉得自己更聪明”，而看下面这些：

- 真题回放命中率是否提升
- 强原文题型的可出题材料命中率是否提升
- 题卡错配率是否下降
- 继续复核比例是否下降
- 用户人工改题幅度是否下降
- 下载转化率是否上升

## 7. 先做什么

从实施角度，最适合你现在马上开始的不是“训模型”，而是三件事：

1. 建真题金标准表
2. 先挑 `main_idea` 和 `sentence_fill` 各 100-300 题做试点
3. 用同一批题跑“材料处理 -> 检索映射 -> 成题 -> 审核评分”的完整回放

后续每一轮改动，都只在这批固定试点集上比前后。
